## 二、流程一 · 策展人创建活动旅程

**核心目标**：从“一个想法”到“可分享的活动链接”，目标 30 秒完成

| 步骤 | 节点类型     | 核心操作          | 关键说明                                                                                   |
| -- | -------- | ------------- | -------------------------------------------------------------------------------------- |
| 01 | 🟢 用户操作  | 打开 HereNow 首页 | 首页展示活动模板画廊 + AI 创建入口                                                                   |
| 02 | 🌸 分支决策  | 选择创建方式        | 🅰️ 一句话描述（核心）：输入自然语言（如“下周六在朝阳公园办野餐读书会”）🅱️ 选择模板：从 40-50 个分类模板选择🅲 上传海报（P2）：多模态 AI 提取信息 |
| 03 | 🔵 AI 驱动 | AI 解析与生成      | NLU 意图解析 → 提取实体 → 匹配模板 → 内容填充 → 风格渲染 → 生成草稿                                            |
| 04 | 🔵 AI 驱动 | 对话式修改         | 支持自然语言指令调整：“标题太正式了”→ 自动更新“加一句新手友好”→ 插入内容“配色更温暖”→ 切换暖色调                                 |
| 05 | 🟢 用户操作  | 预览与微调         | 所见即所得编辑器，手动修改标题、描述、时间、地点、议程等核心字段                                                       |
| 06 | 🟣 系统操作  | 设置报名规则        | 人数上限 · 自定义表单字段 · 审核模式（自动/手动）· 候补开关                                                     |
| 07 | 🟠 核心路径  | 一键发布          | 生成唯一活动链接；自动配置 SEO 元数据、Open Graph 标签、Schema.org 结构化数据                                   |
| 08 | 🟡 增长节点  | 分享传播          | 复制链接分享到微信/WhatsApp/社群/社交媒体；每次分享 = 免费 SEO 外链                                            |

## 三、流程二 · 参与者报名旅程

**核心目标**：零注册摩擦，报名永远不需要 HereNow 账号

| 步骤 | 节点类型    | 核心操作     | 关键说明                                                                          |
| -- | ------- | -------- | ----------------------------------------------------------------------------- |
| 01 | 🟢 用户操作 | 收到活动链接   | 来源：社交媒体/社群消息/搜索引擎                                                             |
| 02 | 🟢 用户操作 | 打开活动页面   | 无需登录，页面加载速度 FCP < 1.5s；首屏可见封面 Banner + 核心信息 + 报名按钮                            |
| 03 | 🟢 用户操作 | 浏览活动详情   | 查看时间、地点、描述、议程、策展人信息、已报名人数                                                     |
| 04 | 🟠 核心路径 | 点击“立即报名” | 移动端：底部悬浮按钮；桌面端：首屏右侧固定按钮                                                       |
| 05 | 🟢 用户操作 | 填写报名表单   | 策展人自定义字段；无需注册 HereNow 账号；邮箱作为隐性身份锚点（Guest 身份）                                 |
| 06 | 🌸 分支决策 | 报名状态判断   | ✅ 名额充足：报名成功 → 确认邮件 + .ics 日历文件⏳ 需要审核：进入审核队列 → 等待策展人确认🔴 已满员：自动加入候补名单 → 有空位时通知 |
| 07 | 🟠 核心路径 | 报名成功页展示  | 展示确认信息；引导升级触点（一键绑定 Google/Apple 账号），转化率 30-40%                                |
| 08 | 🟣 系统操作 | 发送确认邮件   | 含活动详情 + .ics 日历文件；活动前 24h/1h 自动发送提醒                                           |

## 四、流程三 · 活动页面生命周期

**核心目标**：活动页面从“用完即弃”变为永久 SEO 内容资产

### 4.1 活动前 · 报名阶段（🟠 核心路径）

- 封面 Banner / 视觉主题
- 活动标题 + 策展人信息
- 📅 时间 · 📍 地点 · 👥 已报名人数
- 活动详情描述（AI 生成）
- 议程 / 时间线
- \[立即报名] 按钮（移动端悬浮底部）
- 策展人其他活动推荐

### 4.2 活动中 · 进行阶段（🟢 用户操作）

- 实时动态更新（可选）
- 签到功能
- 策展人管理参与者状态（已确认/取消/候补/已签到）

### 4.3 活动后 · 内容沉淀阶段（🟣 系统/🔵 AI 驱动）

- 封面更新为活动现场照，标题添加“已结束”标识
- 📸 活动相册（批量上传）
- 📝 文字回顾（富文本 / AI 辅助生成）
- 📎 课件上传（PDF/PPT 嵌入）
- 🎬 视频链接嵌入
- ⭐ 参与者星级评分 + 文字评价
- 策展人其他活动推荐 + \[关注策展人] 入口

## 五、流程四 · 用户身份渐进升级

**核心目标**：零摩擦入口，渐进式提升用户粘性，无强制注册门槛

### 5.1 三层身份设计

| 身份层级             | 核心权益                                           | 升级触发场景（转化率）                                 |
| ---------------- | ---------------------------------------------- | ------------------------------------------- |
| 👻 Guest（匿名）     | 仅填写表单即可报名，无需注册，数据 100% 归策展人；                   | -（初始状态）                                     |
| 🔑 Lite User（轻量） | 一键绑定 Google/Apple 账号解锁：查看历史报名、关注策展人、接收推送、下载课件； | 报名成功页（30-40%）、活动回顾发布（15-25%）、新活动推送（10-20%）； |
| ⭐ Full User（活跃）  | 完整账号注册解锁：创建活动、使用 AI 功能、个人品牌主页、数据看板；            | Lite User 阶段深度使用后引导；                        |

### 5.2 绝对禁忌（🚫 绝对不做）

- 报名前强制要求注册；
- 浏览活动时弹窗诱导注册；
- 首次打开活动链接强制注册。

## 六、补充说明

### 6.1 通知与沟通系统（核心闭环）

- **报名全流程通知**：报名成功/审核中/候补成功、审核通过/驳回、候补递补（邮件/推送）；
- **活动提醒通知**：活动前 24h（邮件）、活动前 1h（邮件/推送）、活动信息变更通知；
- **内容触达通知**：活动回顾发布、关注策展人新活动、评价/互动反馈。

### 6.2 官网 SEO 基建（增长核心）

- **技术层**：SSR/SSG 渲染（FCP < 1.5s）、自动生成 sitemap、移动端响应式设计；
- **结构化数据**：自动添加 Schema.org（Event 类型）、优化 Open Graph 社交标签；
- **外链增长**：活动分享链接作为免费 SEO 外链，内容矩阵（博客+活动沉淀）覆盖行业关键词。
