# Google 登录功能安装说明

## 前置要求

在开始使用 Google 登录功能之前，需要安装以下依赖：

### 1. 安装 Supabase 依赖

```bash
cd apps/frontend
npm install @supabase/supabase-js @supabase/auth-helpers-nextjs
# 或
pnpm add @supabase/supabase-js @supabase/auth-helpers-nextjs
# 或
yarn add @supabase/supabase-js @supabase/auth-helpers-nextjs
```

### 2. 配置环境变量

在 `apps/frontend/.env.local` 文件中添加以下环境变量：

```env
NEXT_PUBLIC_SUPABASE_URL=your_supabase_project_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key
```

**获取 Supabase 凭证**：
1. 登录 [Supabase Dashboard](https://app.supabase.com)
2. 选择你的项目
3. 进入 Settings → API
4. 复制 `Project URL` 和 `anon public` key

### 3. 配置 Supabase Google OAuth

1. 在 Supabase Dashboard 中，进入 **Authentication** → **Providers**
2. 启用 **Google** Provider
3. 配置 Google OAuth：
   - 在 [Google Cloud Console](https://console.cloud.google.com/) 创建 OAuth 2.0 客户端 ID
   - 添加授权重定向 URI：`https://<your-project-ref>.supabase.co/auth/v1/callback`
   - 将 Client ID 和 Client Secret 填入 Supabase

4. 配置重定向 URL：
   - 开发环境：`http://localhost:3000/auth/callback`
   - 生产环境：`https://yourdomain.com/auth/callback`

### 4. 执行数据库脚本

确保已执行以下 SQL 脚本（按顺序）：

1. `docs/database/sql/01_profiles.sql` - 创建 profiles 表和触发器
2. `docs/database/sql/02_plans.sql` - 创建 plans 表并插入基础套餐
3. `docs/database/sql/03_subscriptions.sql` - 创建 subscriptions 表
4. `docs/database/sql/05_communities.sql` - 创建 communities 表
5. `docs/database/sql/04_events.sql` - 创建 events 表

### 5. 执行 Mock 数据脚本（可选）

如果需要为测试用户创建数据，执行：

```sql
-- 在 Supabase SQL Editor 中执行
-- docs/database/sql/06_mock_data.sql
```

这将为用户 `2b47c0ca-b225-4575-a7cb-441ab9474db6` 创建：
- Profile 记录
- Free Plan 订阅
- 2 个 Communities
- 3 个 Events

## 验证安装

1. **启动开发服务器**：
   ```bash
   cd apps/frontend
   npm run dev
   ```

2. **访问首页**：
   打开 `http://localhost:3000`

3. **测试登录**：
   - 点击导航栏的 "使用 Google 登录" 按钮
   - 应该跳转到 Google 登录页面
   - 登录成功后重定向到 `/auth/callback`
   - 最终跳转到 `/dashboard`

4. **检查数据库**：
   - 在 Supabase Dashboard → Table Editor 查看 `auth.users` 表
   - 确认 `public.profiles` 表中有新用户记录

## 常见问题

### 问题 1: 模块找不到错误

**错误**: `Cannot find module '@supabase/supabase-js'`

**解决**: 运行 `npm install` 安装依赖

### 问题 2: OAuth 回调 404

**错误**: 登录后重定向到 `/auth/callback` 返回 404

**解决**: 
- 确认已创建 `apps/frontend/app/auth/callback/route.ts` 文件
- 确认 Supabase Dashboard 中配置的重定向 URL 正确

### 问题 3: Google OAuth 配置错误

**错误**: `redirect_uri_mismatch`

**解决**: 
- 检查 Google Cloud Console 中的授权重定向 URI
- 确保 URI 与 Supabase 配置的一致
