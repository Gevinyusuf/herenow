## 技术选型理由

### 前端：Next.js
- **服务端渲染**：提升首屏加载速度和 SEO
- **API Routes**：简化前后端集成
- **文件系统路由**：直观的路由管理
- **优化性能**：自动代码分割和图片优化

### 后端：FastAPI
- **高性能**：基于 Starlette 和 Pydantic，性能接近 Node.js
- **类型安全**：Python 类型提示 + Pydantic 验证
- **自动文档**：自动生成 OpenAPI/Swagger 文档
- **异步支持**：原生支持 async/await

### 数据库：Supabase
- **PostgreSQL**：强大的关系型数据库
- **实时功能**：支持实时数据订阅
- **身份认证**：内置 JWT 认证系统
- **REST API**：自动生成 RESTful API

### Monorepo：Turborepo
- **增量构建**：只构建变更的模块
- **并行执行**：多任务并行处理
- **智能缓存**：构建结果缓存，加速 CI/CD
- **依赖管理**：统一的依赖管理

## 数据流

### 1. 用户认证流程

```
用户登录
    ↓
前端调用 Supabase Auth API
    ↓
Supabase 返回 JWT Token
    ↓
前端存储 Token (localStorage/cookie)
    ↓
后续请求携带 Token 到 API 网关
    ↓
API 网关验证 Token 有效性
    ↓
验证通过，转发请求到后端服务
```

### 2. API 请求流程

```
前端发起请求
    ↓
携带 JWT Token
    ↓
API 网关接收请求
    ↓
验证 JWT Token (调用 Supabase 验证)
    ↓
解析用户信息
    ↓
根据路径路由到对应服务
    ↓
后端服务处理业务逻辑
    ↓
查询/更新 Supabase 数据库
    ↓
返回响应
    ↓
API 网关记录日志
    ↓
返回给前端
```

## 安全架构

### 多层安全防护

1. **前端层**
   - HTTPS 传输加密
   - Token 安全存储
   - XSS 防护

2. **API 网关层**
   - JWT Token 验证
   - 请求限流
   - IP 白名单（可选）

3. **服务层**
   - 参数验证
   - SQL 注入防护
   - 权限检查

4. **数据库层**
   - 连接加密
   - 行级安全策略 (RLS)
   - 数据备份

## 扩展性设计

### 水平扩展
- **前端**：无状态设计，可多实例部署
- **API 网关**：负载均衡，多实例运行
- **后端服务**：微服务架构，独立扩展

### 垂直扩展
- 根据服务负载调整资源配置
- 数据库读写分离
- 缓存层优化

## 容错设计

### 服务降级
- API 网关故障时，可配置备用网关
- 服务不可用时，返回友好错误信息

### 重试机制
- 自动重试失败的请求
- 指数退避策略

### 监控告警
- 实时监控服务健康状态
- 异常情况自动告警
