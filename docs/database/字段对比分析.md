# 前端 EventData 与数据库 events_v1 字段对比分析

## ✅ 已存储的字段

| 前端字段 | 数据库字段 | 存储方式 | 状态 |
|---------|-----------|---------|------|
| `eventName` | `title` | 直接映射 | ✅ |
| `coverImage` | `cover_image_url` | 直接映射 | ✅ |
| `startDate` + `startTime` | `start_at` | 组合为 TIMESTAMPTZ | ✅ |
| `endDate` + `endTime` | `end_at` | 组合为 TIMESTAMPTZ | ✅ |
| `selectedTimezone.id` | `timezone` | 直接映射 | ✅ |
| `location` / `isVirtual` / `meetingLink` | `location_info` | JSONB | ✅ |
| `description` | `description` | 直接映射 | ✅ |
| `questions` | `registration_fields` | JSONB 数组 | ✅ |
| `selectedTheme` + `selectedEffect` | `style_config` | JSONB | ✅ |
| `host.id` | `host_id` | UUID 外键 | ✅ |
| `visibility` | `visibility` | 直接映射 | ✅ |
| `requireApproval` | `require_approval` | 直接映射 | ✅ |

## ❌ 遗漏的字段

### 1. **tickets** - 票务配置
- **前端字段**: `tickets: Ticket[]`
- **数据库**: 无对应字段
- **影响**: 无法存储票务信息（票种、价格、数量等）
- **建议**: 添加到 `style_config` 或创建独立的 `ticket_config` JSONB 字段

### 2. **coHosts** - 联合主办方
- **前端字段**: `coHosts: any[]`
- **数据库**: 无对应字段
- **后端**: 未接收此字段
- **影响**: 无法存储联合主办方信息
- **建议**: 创建 `co_hosts` JSONB 字段或关联表

### 3. **isLocationPublic** - 地点是否公开
- **前端字段**: `isLocationPublic: boolean`
- **数据库**: `location_info` 中未包含
- **后端**: 未处理此字段
- **影响**: 无法控制地点信息的可见性（注册后可见 vs 所有人可见）
- **建议**: 添加到 `location_info` JSONB 中

## 🔧 需要修复的内容

1. **后端 API 模型** (`EventCreateRequest`): 添加 `coHosts` 字段
2. **后端存储逻辑**: 
   - 将 `tickets` 存储到 `ticket_config` JSONB 字段
   - 将 `coHosts` 存储到 `co_hosts` JSONB 字段
   - 将 `isLocationPublic` 添加到 `location_info` 中
3. **数据库表结构**: 可能需要添加 `ticket_config` 和 `co_hosts` 字段（或使用现有 JSONB 字段）
