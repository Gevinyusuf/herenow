# events_v1 表字段存储修复说明

## 修复内容

已修复前端 EventData 中遗漏的字段存储问题。

## 遗漏的字段（已修复）

### 1. ✅ tickets - 票务配置
- **前端字段**: `tickets: Ticket[]`
- **数据库字段**: `ticket_config` JSONB
- **存储结构**:
```json
{
  "tickets": [
    {
      "id": 1,
      "name": "General Admission",
      "type": "free",  // 'free' | 'paid'
      "price": "",
      "quantity": "",
      "requireApproval": false
    }
  ]
}
```

### 2. ✅ coHosts - 联合主办方
- **前端字段**: `coHosts: any[]`
- **数据库字段**: `co_hosts` JSONB
- **存储结构**: 直接存储数组
```json
[
  {
    "id": "uuid",
    "name": "Co-host Name",
    "type": "user" | "org",
    ...
  }
]
```

### 3. ✅ isLocationPublic - 地点是否公开
- **前端字段**: `isLocationPublic: boolean`
- **数据库字段**: `location_info.isPublic` (JSONB 中的字段)
- **存储结构**:
```json
{
  "type": "offline",
  "name": "San Francisco",
  "isPublic": false  // 新增字段
}
```

## 数据库迁移

### 对于现有数据库
运行以下 SQL 脚本添加缺失字段：
```sql
-- 文件: docs/database/sql/04_events_v1_add_missing_fields.sql
ALTER TABLE events_v1 
ADD COLUMN IF NOT EXISTS ticket_config JSONB DEFAULT '{"tickets": []}'::jsonb;

ALTER TABLE events_v1 
ADD COLUMN IF NOT EXISTS co_hosts JSONB DEFAULT '[]'::jsonb;
```

### 对于新数据库
`04_events_v1.sql` 已更新，包含所有字段。
