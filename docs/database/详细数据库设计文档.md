# HereNow 详细数据库设计文档

## 1. 设计原则与规范

### 1.1 核心设计原则
- **以用户为中心**：单一账户体系，支持 B 端（主办方）和 C 端（参会者）双重身份
- **Freemium 商业模式**：通过套餐（Plans）和订阅（Subscriptions）控制功能权限
- **扩展性优先**：使用 JSONB 存储可变配置，减少数据库结构变更
- **数据完整性**：通过外键约束、唯一约束、检查约束保证数据一致性
- **性能优化**：合理设计索引，支持高频查询场景

### 1.2 命名规范
- **表名**：小写，复数形式，使用下划线分隔（如 `community_members`）
- **字段名**：小写，使用下划线分隔（如 `created_at`）
- **主键**：统一使用 `id`，类型为 `BIGINT`
- **外键**：使用 `表名_id` 格式（如 `user_id`, `community_id`）
- **时间字段**：统一使用 `TIMESTAMPTZ`（带时区的时间戳）
- **布尔字段**：使用 `is_` 或 `has_` 前缀（如 `is_public`, `is_active`）

### 1.3 数据类型选择
- **主键**：`BIGINT GENERATED ALWAYS AS IDENTITY`（自增，支持大容量）
- **时间戳**：`TIMESTAMPTZ`（带时区，支持全球用户）
- **可变配置**：`JSONB`（支持查询和索引）
- **文本**：`TEXT`（长文本）或 `VARCHAR(n)`（有长度限制的文本）
- **金额**：`DECIMAL(10, 2)`（精确到分）
- **枚举值**：`VARCHAR(n)` 配合应用层约束

---

## 2. 数据库架构概览

### 2.1 模块划分
1. **身份与权限模块**：用户、套餐、订阅
2. **内容与社群模块**：社群、活动
3. **交互与数据模块**：成员关系、报名记录
4. **扩展模块**：票务、问卷、支付（未来扩展）

### 2.2 核心实体关系
```
Users (1) ──< (N) Subscriptions
Users (1) ──< (N) Communities (作为 Owner)
Users (N) ──< (N) Communities (通过 CommunityMembers)
Communities (1) ──< (N) Events
Events (1) ──< (N) Registrations
Users (1) ──< (N) Registrations
```

## 3. 详细表结构设计

### 3.1 身份与权限模块

#### 3.1.1 `users` 表（用户核心表）

**表说明**：存储用户基础信息，支持单一账户体系。

**字段设计**：

| 字段名 | 数据类型 | 约束 | 说明 |
|--------|---------|------|------|
| `id` | `BIGINT` | PRIMARY KEY, IDENTITY | 用户唯一标识 |
| `email` | `VARCHAR(255)` | UNIQUE, NOT NULL | 邮箱地址（登录账号） |
| `password_hash` | `VARCHAR(255)` | NULL | 密码哈希（如使用第三方认证可为空） |
| `full_name` | `VARCHAR(100)` | NULL | 用户全名 |
| `avatar_url` | `TEXT` | NULL | 头像 URL |
| `primary_intent` | `VARCHAR(20)` | DEFAULT 'HYBRID' | 用户主要意图：'CONSUMER'（参会者）、'ORGANIZER'（主办方）、'HYBRID'（混合） |
| `created_at` | `TIMESTAMPTZ` | DEFAULT NOW() | 创建时间 |
| `updated_at` | `TIMESTAMPTZ` | DEFAULT NOW() | 更新时间 |

**索引设计**：
- `idx_users_email`：`email` 字段唯一索引（登录查询）
- `idx_users_created_at`：`created_at` 字段索引（用户统计查询）

**业务规则**：
- `email` 必须唯一，作为登录凭证
- `primary_intent` 仅用于前端路由引导，不影响实际权限
- `updated_at` 应在每次更新时自动更新（通过触发器或应用层）

---

#### 3.1.2 `plans` 表（套餐定义表）

**表说明**：定义不同套餐的权益和限制，支持 Freemium 模式。

**字段设计**：

| 字段名 | 数据类型 | 约束 | 说明 |
|--------|---------|------|------|
| `id` | `INT` | PRIMARY KEY | 套餐 ID（1=Free, 2=Pro, 3=Enterprise） |
| `name` | `VARCHAR(50)` | NOT NULL | 套餐名称 |
| `price_monthly` | `DECIMAL(10, 2)` | DEFAULT 0 | 月价格（免费版为 0） |
| `price_yearly` | `DECIMAL(10, 2)` | NULL | 年价格（可选） |
| `limits` | `JSONB` | NOT NULL, DEFAULT '{}' | 权益配置 JSON，包含：<br>- `max_communities`: 最大社群数<br>- `max_events_per_month`: 每月最大活动数<br>- `max_event_capacity`: 单个活动最大容量<br>- `remove_branding`: 是否移除品牌标识<br>- `can_export_data`: 是否可导出数据<br>- `can_custom_domain`: 是否支持自定义域名<br>- `support_priority`: 支持优先级 |
| `features` | `JSONB` | DEFAULT '{}' | 功能特性 JSON，包含功能开关 |
| `is_active` | `BOOLEAN` | DEFAULT TRUE | 是否启用 |
| `display_order` | `INT` | DEFAULT 0 | 显示顺序 |
| `created_at` | `TIMESTAMPTZ` | DEFAULT NOW() | 创建时间 |
| `updated_at` | `TIMESTAMPTZ` | DEFAULT NOW() | 更新时间 |

**索引设计**：
- `idx_plans_is_active`：`is_active` 字段索引（查询可用套餐）
- `idx_plans_display_order`：`display_order` 字段索引（排序显示）

**JSONB 字段示例**：
```json
{
  "max_communities": 1,
  "max_events_per_month": 3,
  "max_event_capacity": 100,
  "remove_branding": false,
  "can_export_data": false,
  "can_custom_domain": false,
  "support_priority": "standard"
}
```

**业务规则**：
- `id` 使用固定整数值，便于应用层判断
- `limits` 字段是权限控制的核心，应用层需严格校验
- `is_active` 为 FALSE 时，新用户不可订阅，但已有订阅不受影响

---

#### 3.1.3 `subscriptions` 表（订阅关系表）

**表说明**：记录用户的订阅关系，一个用户同一时间只能有一个有效订阅。

**字段设计**：

| 字段名 | 数据类型 | 约束 | 说明 |
|--------|---------|------|------|
| `id` | `BIGINT` | PRIMARY KEY, IDENTITY | 订阅记录 ID |
| `user_id` | `BIGINT` | UNIQUE, NOT NULL, REFERENCES users(id) | 用户 ID |
| `plan_id` | `INT` | NOT NULL, REFERENCES plans(id) | 套餐 ID |
| `status` | `VARCHAR(20)` | NOT NULL, DEFAULT 'active' | 订阅状态：'active'（生效）、'canceled'（已取消）、'past_due'（逾期）、'trialing'（试用中）、'expired'（已过期） |
| `current_period_start` | `TIMESTAMPTZ` | NULL | 当前计费周期开始时间 |
| `current_period_end` | `TIMESTAMPTZ` | NULL | 当前计费周期结束时间 |
| `canceled_at` | `TIMESTAMPTZ` | NULL | 取消时间 |
| `gateway_subscription_id` | `VARCHAR(100)` | NULL | 支付网关订阅 ID（Stripe/LemonSqueezy） |
| `gateway_customer_id` | `VARCHAR(100)` | NULL | 支付网关客户 ID |
| `trial_ends_at` | `TIMESTAMPTZ` | NULL | 试用结束时间 |
| `created_at` | `TIMESTAMPTZ` | DEFAULT NOW() | 创建时间 |
| `updated_at` | `TIMESTAMPTZ` | DEFAULT NOW() | 更新时间 |

**索引设计**：
- `idx_subscriptions_user_id`：`user_id` 字段唯一索引（已通过 UNIQUE 约束实现）
- `idx_subscriptions_status`：`status` 字段索引（查询有效订阅）
- `idx_subscriptions_period_end`：`current_period_end` 字段索引（到期提醒查询）

**业务规则**：
- `user_id` 唯一约束确保一个用户只有一个有效订阅
- `status` 为 'active' 或 'trialing' 时，用户享有套餐权益
- 新用户注册时自动创建免费套餐订阅（plan_id=1）
- `current_period_end` 过期后，`status` 应自动更新为 'expired'（通过定时任务）

### 3.2 内容与社群模块

#### 3.2.1 `communities` 表（社群/组织者主页）

**表说明**：社群是活动的容器，每个活动必须归属于一个社群。

**字段设计**：

| 字段名 | 数据类型 | 约束 | 说明 |
|--------|---------|------|------|
| `id` | `BIGINT` | PRIMARY KEY, IDENTITY | 社群 ID |
| `owner_id` | `BIGINT` | NOT NULL, REFERENCES users(id) | 创建者/所有者 ID |
| `name` | `VARCHAR(100)` | NOT NULL | 社群名称 |
| `description` | `TEXT` | NULL | 社群描述 |
| `slug` | `VARCHAR(30)` | UNIQUE, NOT NULL | URL 友好标识符 |
| `logo_url` | `TEXT` | NULL | Logo 图片 URL |
| `cover_image_url` | `TEXT` | NULL | 封面图片 URL |
| `settings` | `JSONB` | DEFAULT '{}' | 社群配置 JSON，包含：<br>- `require_approval`: 加入是否需要审核<br>- `is_public`: 是否公开可见<br>- `contact_email`: 联系方式<br>- `social_links`: 社交媒体链接<br>- `custom_fields`: 自定义字段 |
| `member_count` | `INT` | DEFAULT 0 | 成员数量（冗余字段，用于快速查询） |
| `event_count` | `INT` | DEFAULT 0 | 活动数量（冗余字段） |
| `created_at` | `TIMESTAMPTZ` | DEFAULT NOW() | 创建时间 |
| `updated_at` | `TIMESTAMPTZ` | DEFAULT NOW() | 更新时间 |

**索引设计**：
- `idx_communities_owner_id`：`owner_id` 字段索引（查询用户创建的社群）
- `idx_communities_slug`：`slug` 字段唯一索引（已通过 UNIQUE 约束实现）
- `idx_communities_created_at`：`created_at` 字段索引（排序查询）

**业务规则**：
- `slug` 必须唯一，用于生成友好的 URL
- `owner_id` 对应的用户自动成为 `community_members` 表中的 OWNER
- `member_count` 和 `event_count` 通过触发器或应用层维护，保持一致性
- 删除社群前需检查是否有关联的活动（通过外键约束或应用层逻辑）

---

#### 3.2.2 `events` 表（活动表）

**表说明**：核心业务实体，存储活动信息。

**字段设计**：

| 字段名 | 数据类型 | 约束 | 说明 |
|--------|---------|------|------|
| `id` | `BIGINT` | PRIMARY KEY, IDENTITY | 活动 ID |
| `community_id` | `BIGINT` | NOT NULL, REFERENCES communities(id) | 所属社群 ID |
| `creator_id` | `BIGINT` | NOT NULL, REFERENCES users(id) | 创建者 ID |
| `title` | `VARCHAR(255)` | NOT NULL | 活动标题 |
| `slug` | `VARCHAR(50)` | UNIQUE, NOT NULL | URL 友好标识符 |
| `description` | `TEXT` | NULL | 活动描述（支持富文本） |
| `cover_image_url` | `TEXT` | NULL | 封面图片 URL |
| `start_time` | `TIMESTAMPTZ` | NOT NULL | 开始时间 |
| `end_time` | `TIMESTAMPTZ` | NOT NULL | 结束时间 |
| `timezone` | `VARCHAR(50)` | NULL | 时区（如 'Asia/Shanghai'） |
| `status` | `VARCHAR(20)` | DEFAULT 'DRAFT' | 状态：'DRAFT'（草稿）、'PUBLISHED'（已发布）、'CANCELED'（已取消）、'ENDED'（已结束） |
| `location_config` | `JSONB` | NULL | 地点配置 JSON，包含：<br>- `type`: 'online'（线上）或 'offline'（线下）或 'hybrid'（混合）<br>- `address`: 线下地址<br>- `online_link`: 线上会议链接<br>- `venue_name`: 场地名称<br>- `coordinates`: 经纬度坐标 |
| `capacity` | `INT` | DEFAULT 100 | 最大容量 |
| `registration_count` | `INT` | DEFAULT 0 | 已报名数量（冗余字段） |
| `is_public` | `BOOLEAN` | DEFAULT TRUE | 是否公开可见 |
| `require_approval` | `BOOLEAN` | DEFAULT FALSE | 报名是否需要审核 |
| `registration_start` | `TIMESTAMPTZ` | NULL | 报名开始时间 |
| `registration_end` | `TIMESTAMPTZ` | NULL | 报名结束时间 |
| `ticket_config` | `JSONB` | DEFAULT '{}' | 票务配置 JSON（详见扩展说明） |
| `question_config` | `JSONB` | DEFAULT '{}' | 问卷配置 JSON（详见扩展说明） |
| `theme_config` | `JSONB` | DEFAULT '{}' | 主题配置 JSON（颜色、样式等） |
| `created_at` | `TIMESTAMPTZ` | DEFAULT NOW() | 创建时间 |
| `updated_at` | `TIMESTAMPTZ` | DEFAULT NOW() | 更新时间 |

**索引设计**：
- `idx_events_community_id`：`community_id` 字段索引（查询社群下的活动）
- `idx_events_creator_id`：`creator_id` 字段索引（查询用户创建的活动）
- `idx_events_slug`：`slug` 字段唯一索引（已通过 UNIQUE 约束实现）
- `idx_events_status_public`：`(status, is_public, start_time)` 复合索引（发现页查询）
- `idx_events_start_time`：`start_time` 字段索引（时间范围查询）
- `idx_events_registration_period`：`(registration_start, registration_end)` 复合索引（报名时间查询）

**检查约束**：
- `end_time > start_time`：结束时间必须晚于开始时间
- `registration_end IS NULL OR registration_end >= registration_start`：报名结束时间必须晚于开始时间

**JSONB 字段示例**：

`location_config`:
```json
{
  "type": "offline",
  "address": "北京市朝阳区xxx",
  "venue_name": "会议中心A厅",
  "coordinates": {
    "lat": 39.9042,
    "lng": 116.4074
  }
}
```

`ticket_config`:
```json
{
  "tickets": [
    {
      "id": 1,
      "name": "早鸟票",
      "type": "paid",
      "price": 99.00,
      "quantity": 50,
      "sold": 0,
      "require_approval": false
    },
    {
      "id": 2,
      "name": "普通票",
      "type": "free",
      "price": 0,
      "quantity": 100,
      "sold": 0,
      "require_approval": false
    }
  ]
}
```

`question_config`:
```json
{
  "questions": [
    {
      "id": "name",
      "label": "姓名",
      "type": "text",
      "required": true,
      "fixed": true
    },
    {
      "id": "email",
      "label": "邮箱",
      "type": "email",
      "required": true,
      "fixed": true
    },
    {
      "id": "custom_1",
      "label": "公司名称",
      "type": "text",
      "required": false,
      "fixed": false
    }
  ]

```

**业务规则**：
- `slug` 必须唯一，用于生成友好的 URL
- `status` 为 'PUBLISHED' 时，活动才对外可见
- `registration_count` 通过触发器或应用层维护，不能超过 `capacity`
- `end_time` 必须晚于 `start_time`
- 活动取消后，相关报名记录状态需同步更新

### 3.3 交互与数据模块

#### 3.3.1 `community_members` 表（社群成员表）

**表说明**：连接用户和社群的多对多关系表，记录成员角色和加入信息。

**字段设计**：

| 字段名 | 数据类型 | 约束 | 说明 |
|--------|---------|------|------|
| `id` | `BIGINT` | PRIMARY KEY, IDENTITY | 成员关系 ID |
| `community_id` | `BIGINT` | NOT NULL, REFERENCES communities(id) | 社群 ID |
| `user_id` | `BIGINT` | NOT NULL, REFERENCES users(id) | 用户 ID |
| `role` | `VARCHAR(20)` | DEFAULT 'MEMBER' | 角色：'OWNER'（所有者）、'ADMIN'（管理员）、'MEMBER'（普通成员） |
| `source` | `VARCHAR(50)` | NULL | 加入来源：'ORGANIC'（主动加入）、'EVENT_REGISTRATION'（报名自动加入）、'INVITED'（被邀请） |
| `invited_by` | `BIGINT` | NULL, REFERENCES users(id) | 邀请人 ID（如果来源为 'INVITED'） |
| `joined_at` | `TIMESTAMPTZ` | DEFAULT NOW() | 加入时间 |
| `last_active_at` | `TIMESTAMPTZ` | NULL | 最后活跃时间 |

**索引设计**：
- `idx_community_members_community_id`：`community_id` 字段索引（查询社群成员）
- `idx_community_members_user_id`：`user_id` 字段索引（查询用户加入的社群）
- `idx_community_members_unique`：`(community_id, user_id)` 联合唯一索引（防止重复加入）

**业务规则**：
- `(community_id, user_id)` 联合唯一，确保一个用户在一个社群中只有一条记录
- `role` 为 'OWNER' 的用户通常是社群的创建者（`communities.owner_id`）
- 用户报名活动时，如果尚未加入该活动的社群，自动创建一条 `source='EVENT_REGISTRATION'` 的记录
- 删除社群时，级联删除所有成员记录（通过外键约束）

---

#### 3.3.2 `registrations` 表（报名/订单表）

**表说明**：记录用户对活动的报名信息，是 C 端用户的核心转化数据。

**字段设计**：

| 字段名 | 数据类型 | 约束 | 说明 |
|--------|---------|------|------|
| `id` | `BIGINT` | PRIMARY KEY, IDENTITY | 报名记录 ID |
| `event_id` | `BIGINT` | NOT NULL, REFERENCES events(id) | 活动 ID |
| `user_id` | `BIGINT` | NOT NULL, REFERENCES users(id) | 用户 ID |
| `status` | `VARCHAR(20)` | DEFAULT 'CONFIRMED' | 状态：'PENDING'（待支付/待审核）、'CONFIRMED'（已确认）、'CANCELED'（已取消）、'CHECKED_IN'（已签到）、'NO_SHOW'（未到场） |
| `ticket_id` | `INT` | NULL | 票务类型 ID（对应 `events.ticket_config` 中的 ticket id） |
| `ticket_price` | `DECIMAL(10, 2)` | DEFAULT 0 | 票务价格（冗余字段，记录报名时的价格） |
| `check_in_code` | `VARCHAR(20)` | NULL, UNIQUE | 签到码/核销码 |
| `check_in_at` | `TIMESTAMPTZ` | NULL | 签到时间 |
| `canceled_at` | `TIMESTAMPTZ` | NULL | 取消时间 |
| `cancel_reason` | `TEXT` | NULL | 取消原因 |
| `answers` | `JSONB` | DEFAULT '{}' | 问卷答案 JSON（对应 `events.question_config`） |
| `payment_id` | `VARCHAR(100)` | NULL | 支付订单 ID（如使用第三方支付） |
| `payment_status` | `VARCHAR(20)` | NULL | 支付状态：'pending'、'paid'、'refunded'、'failed' |
| `created_at` | `TIMESTAMPTZ` | DEFAULT NOW() | 创建时间 |
| `updated_at` | `TIMESTAMPTZ` | DEFAULT NOW() | 更新时间 |

**索引设计**：
- `idx_registrations_event_id`：`event_id` 字段索引（查询活动的报名记录）
- `idx_registrations_user_id`：`user_id` 字段索引（查询用户的报名记录）
- `idx_registrations_status`：`status` 字段索引（按状态查询）
- `idx_registrations_check_in_code`：`check_in_code` 字段唯一索引（签到查询）
- `idx_registrations_created_at`：`created_at` 字段索引（时间排序）
- `idx_registrations_event_user`：`(event_id, user_id)` 复合索引（防止重复报名）

**业务规则**：
- `(event_id, user_id)` 应唯一，防止重复报名（通过应用层或唯一索引实现）
- `check_in_code` 必须唯一，用于签到核销
- `status` 为 'CONFIRMED' 时，才计入活动的 `registration_count`
- `ticket_price` 记录报名时的价格，即使后续价格变更也不影响
- `answers` 存储用户填写的问卷答案，格式需与 `events.question_config` 对应
- 报名成功后，自动将用户加入对应社群（如果尚未加入）

## 4. 扩展表设计（未来功能）

### 4.1 `tickets` 表（独立票务表，可选）

如果票务系统需要更复杂的逻辑（如库存管理、价格变动历史），可考虑将票务从 `events.ticket_config` JSONB 中提取为独立表。

**字段设计**：

| 字段名 | 数据类型 | 约束 | 说明 |
|--------|---------|------|------|
| `id` | `BIGINT` | PRIMARY KEY, IDENTITY | 票务 ID |

## 5. 索引策略总结

### 5.1 主键索引
所有表的主键自动创建唯一索引。

### 5.2 外键索引
所有外键字段创建索引，支持关联查询和级联操作。

### 5.3 唯一索引
- `users.email`
- `communities.slug`
- `events.slug`
- `subscriptions.user_id`
- `community_members(community_id, user_id)`
- `registrations.check_in_code`

### 5.4 复合索引
- `events(status, is_public, start_time)`：发现页查询
- `events(registration_start, registration_end)`：报名时间查询
- `registrations(event_id, user_id)`：防止重复报名

### 5.5 查询优化索引
- `users.created_at`：用户统计
- `events.start_time`：时间范围查询
- `registrations.created_at`：报名统计
- `subscriptions.status`：有效订阅查询

## 6. 数据完整性约束

### 6.1 外键约束
- 所有外键字段设置 `REFERENCES` 约束
- 删除策略：
  - `communities` 删除时，级联删除 `events` 和 `community_members`
  - `events` 删除时，级联删除 `registrations`
  - `users` 删除时，需先处理关联数据（建议软删除）

### 6.2 唯一约束
- `users.email`：邮箱唯一
- `communities.slug`：社群 slug 唯一
- `events.slug`：活动 slug 唯一
- `subscriptions.user_id`：用户唯一订阅
- `community_members(community_id, user_id)`：防止重复加入
- `registrations.check_in_code`：签到码唯一

### 6.3 检查约束
- `events.end_time > start_time`：结束时间晚于开始时间
- `events.registration_end >= registration_start`：报名时间合理性
- `events.capacity > 0`：容量必须大于 0
- `events.registration_count <= capacity`：报名数不超过容量

## 7. 性能优化建议

### 7.1 查询优化
- 使用覆盖索引减少回表查询
- 合理使用 JSONB 索引（GIN 索引）支持 JSON 字段查询
- 避免全表扫描，确保 WHERE 条件使用索引

### 7.2 数据冗余
- `communities.member_count`、`communities.event_count`：减少 COUNT 查询
- `events.registration_count`：快速获取报名数量
- `registrations.ticket_price`：记录历史价格

### 7.3 分区策略（未来）
- `registrations` 表可按 `created_at` 按月分区
- `events` 表可按 `start_time` 按年分区

### 7.4 缓存策略
- 套餐信息（`plans`）可缓存
- 用户订阅信息（`subscriptions`）可缓存
- 热门活动的报名统计可缓存

### 8.1 迁移策略
- 使用数据库迁移工具（如 Prisma Migrate、Alembic）

- 删除字段前，先标记为废弃，再逐步迁移数据

### 9.2 访问控制

- 应用层实现基于角色的权限控制

- 定期全量备份
- 实时增量备份
- 备份数据加密存储
