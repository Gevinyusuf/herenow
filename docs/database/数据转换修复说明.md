# 前端数据与数据库对应关系修复说明

## 修复内容

已修复前端创建活动页面数据与数据库 `events_v1` 表结构的对应关系问题。

- 更新 `handleSubmit()` 函数，将 `isLocationPublic` 包含在提交数据中

## 数据转换映射

### ✅ 直接对应字段
- `eventName` → `title`
- `coverImage` → `cover_image_url`
- `description` → `description`
- `visibility` → `visibility`
- `requireApproval` → `require_approval`
- `host.id` → `host_id`

### ⚠️ 转换后的字段

#### 1. 时间字段
```typescript
// 前端: startDate + startTime → 数据库: start_at (TIMESTAMPTZ)
start_at: combineDateTime(startDate, startTime, timezone)
end_at: combineDateTime(endDate, endTime, timezone)
timezone: selectedTimezone.id
```

#### 2. 地点信息 (location_info JSONB)
```typescript
// 虚拟活动
{
  type: "virtual",
  link: meetingLink
}

// 线下活动
{
  type: "offline",
  name: location,
  isPublic: isLocationPublic || false
}
```

#### 3. 主题配置 (style_config JSONB)
```typescript
{
  themeId: selectedTheme.id,
  effect: selectedEffect,
  colors: {
    bg: selectedTheme.bg,
    contentBg: selectedTheme.contentBg,
    text: selectedTheme.text,
    button: selectedTheme.button
  },
  preview: selectedTheme.preview
}
```

#### 4. 报名表单 (registration_fields JSONB)
```typescript
questions.map(q => ({
  id: q.id,
  label: q.label,
  required: q.required,
  fixed: q.fixed
}))
```

#### 5. 票务配置 (ticket_config JSONB)
```typescript
{
  tickets: tickets.map(t => ({
    id: t.id,
    name: t.name,
    type: t.type,
    price: t.type === 'paid' ? parseFloat(t.price) || 0 : 0,
    quantity: t.quantity ? parseInt(t.quantity) : null,
    requireApproval: t.requireApproval
  }))
}
```

#### 6. Slug 生成
```typescript
// 从 eventName 生成唯一 slug
slug = generateSlug(eventName)
// 例如: "Product Launch 2025" → "product-launch-2025-lx8k2m"
```

## 注意事项

### 1. 时区处理
- 当前实现使用本地时间创建 Date 对象
- 如果需要更精确的时区处理，建议使用 `date-fns-tz` 或类似库

### 2. 地点经纬度
- 当前只存储地点名称
- 如果需要经纬度，需要从 Google Places API 获取并添加到 `location_info` 中

### 3. Tickets 和 CoHosts
- 这些字段存储在 JSONB 中
- 如果后端需要独立表，需要相应调整

### 4. Slug 唯一性
- 当前使用时间戳确保唯一性
- 如果后端需要检查唯一性，可能需要重试机制
